## 根因分类（生成新课程时逐条检查）

### 类型A：题库设计 — 信息泄露
- [ ] B选项是否有图片暴露答案？（B应该只看英文文字，不看图）
- [ ] A选的中文信息+B的选项图片，能不能不懂英文就猜对？
- [ ] 中文描述是否太直白？（"会飞"→只有bird会飞）

### 类型B：题库设计 — 信息差不足
- [ ] A会了第一题，B是否不需要额外知识就能答第二题？
- [ ] AB两步之间是否有真正的认知挑战？

### 类型C：题库设计 — 选项问题
- [ ] 所有选择题选项是否>=4个？（排除判断题）
- [ ] 选项之间是否有足够干扰性？（不能一眼排除）

### 类型D：UI/交互
- [ ] overlay是否全屏覆盖（position:fixed）？
- [ ] 按钮状态变化是否正确（灰→绿→深绿→已提交）？
- [ ] 选项随机打乱了吗？

### 类型E：数据
- [ ] 飞书字段类型是否匹配？（数字→Number，单选→字符串）
- [ ] 不发null值？
- [ ] 版本号是否bump？

### 类型F：支架
- [ ] 支架内容对A和B都有用吗？（A英文支架，B中文/图片支架）
- [ ] 支架是否"减负不减任务"？（不能直接给答案）

## BUG-021：情景阅读题英文句子不可见 + 界面卡死

**状态：🔧 代码已修，待验证**

**现象**：iPhone 6 Plus 上，coop_read_scenario 题型：
1. 英文句子（.coop-read-text）字体太小（18px），学生看不清
2. 界面看似卡死（B区选项被裁掉，scrollIntoView 在 overflow:hidden 容器失效）

**根因**：
- `.content-area` 全局设置 `overflow: hidden`，在 iPhone 上不可滚动，B区选项被裁
- `.coop-read-text` 字号 18px，低于 CLAUDE.md 要求的 32px+

**修复**：
- `style.css`: `.content-area` 改为 `overflow-y: auto`（可滚动）
- `style.css`: `.coop-read-text` 字号从 18px 升至 26px，加 `width: 100%`
- 更新测试、CHANGELOG、bug-patterns

**验证方法**：
1. 进入 U1L1 → 阅读 → 情境阅读（第4站）
2. iPhone 6 Plus 真机测试：英文句子是否清晰可见（26px）
3. 选择A区答案 → 按确认 → B区是否出现并可滚动到 → B区选项是否可点击
