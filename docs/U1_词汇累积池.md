## 情景剧可行性判断（三要素逐项）

### 1. 问句可用性 🟢 可用
- `Is it a ___?` 是 U1L3 的核心句型，完全可用于"猜动物"剧情
- `What's that?` chant 里出现过，可用于"指认惊讶"场景
- ❌ 但没有 `What color is it?` / `How many?` / `Where?` 等常见问句（这些要等 U2/U3）

### 2. 寒暄词可用性 🟢 勉强可用
- Hello / Hi（L3）/ Good afternoon（L4）可以开场
- Yes, it is. / No, it isn't.（L3）可做应答
- Cool! / Great!（L2）可做赞叹
- ❌ 仍没有 Thank you / You're welcome / Goodbye / Bye（这些要等 U2L2/L3）

### 3. 对话轮次可行性 🟢 4-6 轮可达
- 有问句 + 应答 + 描述 + 感叹 → 可以构造真正的多轮互动
- 可行对话模板示例：
  ```
  A: Hello! Look! This is an animal.       (L1+L3)
  B: Cool! What's that?                     (L2+L2)
  A: Guess! It has long ears. It can jump. (L3)
  B: Is it a rabbit?                        (L3)
  A: Yes, it is! It's white and cute.      (L3+L1)
  B: Great! Let's draw it together!         (L2+L4)
  ```
  共 6 轮，全部落在 L1-L4 累积池内。

### 最终结论：🟢 **U1L4 理论上支持 project-based 合作情景剧**

**前提**：必须先把 U1L2 / U1L3 / U1L4 的知识点清单和题库建完；在实际题库产出前，该结论只是"课本上限估算"，不是实锤。

**对比 U1L1**：
- U1L1 独立做情景剧 → 🔴 完全不可做（0 问句 0 寒暄）
- U1L4 累积后做情景剧 → 🟢 可做（有 Is it...? + Hello + Let's + Yes/No）

## 备选方案与风险提示

### 如果不想等 U1L2/L3/L4 全建完
- **最早可做 project-based 情景剧的点**：U1L3 之后（因为 Is it ___? 和 Hello/Yes/No 在 L3 才首次出现）
- U1L2 结尾其实只有 Let's + What's that? + Cool，问答对话还不成立
- **推迟到 U2L4 的好处**：U2 会学 Thank you / You're welcome / Whose / Where is / in/on/under → 对话更自然
- **推迟到 U3L4 的好处**：增加 What's the time / What are you doing / 现在进行时，剧情可扩展到"日常活动"而不只"猜动物"

### 风险提示（给后续 agent）
1. 本文件的 L2-L4 数据**全部来自课本原文**，未经题库实现、未经白名单审核
2. 真实题库产出后必须用与 U1L1 相同的方法重新审计（白名单交叉 + 题库 grep）
3. 在 U1L2/L3/L4 知识点清单建完之前，**不要**基于本文件写死情景剧台词
4. 情景剧里任何超出 L1-L4 累积池的词都算超纲；尤其注意：U1 全单元没有 `What color` / `How many` / `Where` / `Thank you` / `Goodbye`
