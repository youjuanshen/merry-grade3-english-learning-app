### 4.4 发现的新问题（Phase 5 专项）

#### 🟡 中-1：behavior log 每题只写 A 一行，没有 B 的独立行

- **位置**：`data-reporter.js:225-249`（buildBehaviorRecord）+ `adventure.js:588-591`（recordQuestionDetail 调用点）
- **代码**：`buildBehaviorRecord` 硬编码 `studentId = roleAStudentId`、`asRoleA = 'true'`；`recordQuestionDetail` 每题只 enqueue 一次。
- **问题**：Obj3（参与度）/ Obj4（学习成果）需要按**个体**追踪每个学生每题的 retryCount / scaffoldLevelUsed / timeSpentMs。当前方案每题只有 A 一行，B 的个人行为只能通过 partnerId 反推，不是直接观测值。
- **影响**：影响研究数据的个体可追溯性，不阻断 App 使用。
- **建议修复**：`recordQuestionDetail` 里 enqueue 两条记录（studentId=A/asRoleA=true、studentId=B/asRoleA=false），或引入单独的"配对汇总行"替代。

#### 🟡 中-2：pair_collab_log 的 `avgCollabDurationMs` 恒为 0

- **位置**：`data-reporter.js:297`：`avgCollabDurationMs: sp.avgCollabDurationMs || 0`
- **问题**：`adventureState.stationProgress` 在整个 `adventure.js` 中**从未**写入 `avgCollabDurationMs` 字段（grep 零命中），所以每条 pair 记录这一列永远是 0。
- **影响**：飞书里这一列误导；教师 Dashboard 若显示"合作时长"会看不到数据。
- **建议修复**：在 `onStepBComplete` 里用 `Date.now() - window.questionStartTime` 累加到 `sp.totalDurationMs`，完成时除以 `(aTotal+bTotal)` 得到平均值。

#### 🟢 低-1：`asRoleA` / `isCorrect` 字段值是字符串 `'true' / 'false'`

- **位置**：`data-reporter.js:236-237`
- **问题**：若飞书 schema 使用复选框（boolean）字段类型，应传 JS 布尔值；若是 text 字段则 OK。需要与飞书表定义对齐一次。

#### 🟢 低-2：空 records flush 语义混乱（不影响正确性）

- **位置**：`data-reporter.js:99`（`if (records.length === 0) return;` 提前 return）+ `tencent_scf.js:266-268`（空队列返回 code:-1）
- **说明**：前端空队列不会提交，理论上不会触发后端 -1。仅为语义不一致，非 bug。

#### 🟢 低-3：_drStorageSize 阈值 150KB 与 setItem 异步 flush 竞态

- **位置**：`data-reporter.js:43-44`
- **说明**：`flushBehaviorQueue` 是异步 Promise，当前 setItem 可能在 flush 返回前就撞上限。现实场景（单节课 ~15KB）远低于阈值，不会触发；兜底逻辑有理论隐患。

## 8. 课堂练习空行根因分析（2026-04-12 补充）

**现象**：飞书 TABLE_SCORES 中存在一行 2026-04-11 17:43 的记录：姓名空、总分0、满分0。

**调查结论：B — 测试残留（agent 跑半截留下的），不修代码。**

**依据**：
- `app.js` L260 调用 `scfPost(payload)`，payload 包含 `studentName/score/maxScore` 等字段，与 `tencent_scf.js` L337 的 TABLE_SCORES 字段映射完全对齐（无字段漂移）。
- tencent_scf.js L336-348 写 TABLE_SCORES 的所有字段均有 `|| 0` / `|| ''` 默认值兜底；当 `studentName=''`、`score=0`、`maxScore=0` 时会写出空行。
- app.js L260 调用点在 `submitResults()` 函数内，只有用户真正完成一轮测试才会到达；但 agent 在 2026-04-11 自动化测试期间可能触发了"不完整的一轮"（两名学生只有一人或没有正确赋值），导致 studentName 为空字符串、分数为 0 的 payload 被提交。
- 字段层无漂移：`payload.studentName → "姓名"`、`payload.score → "总分"`、`payload.maxScore → "满分"` 映射均正确。

**操作建议**：直接在飞书手动删除该空行。无需改代码。
