## 4. 数据链路

```
学生按下 🎤
  ↓
coop-types.js: attachSoeRecordButton 启动 SpeechEvaluator.beginRecordAndEvaluate(refText)
  ↓
soe-evaluator.js: getUserMedia → ScriptProcessor → Float32 chunks
  ↓
学生再次按下 → controller.stop()
  ↓
resample 16kHz → WAV → Base64
  ↓
POST /api/speech-eval { text: refText, audio: base64 }
  ↓
_server.js → callTencentSOE → wss://soe.cloud.tencent.com/...
  ↓
evalResult → { soeScore, soePronAccuracy, soePronFluency, soePronCompletion }
  ↓
写到 window._pendingSoeScores[role='A'|'B']
  ↓
onStepXComplete → adventure.js onStepBComplete → recordQuestionDetail
  ↓
data-reporter.js buildBehaviorRecord: 从 _pendingSoeScores.A/B 取值分别注入 recA/recB
  ↓
enqueueBehaviorLog → flushBehaviorQueue → scfPost({ action: 'logBehavior', records })
  ↓
tencent_scf.js: batchCreate 到飞书 app_behavior_log
```

## 5. 字段对齐（⚠️ 与任务描述有差异）

**任务描述字段名**：`accuracy / fluency / completion / pronAccuracy`
**ADR-006 权威 schema**：`soeScore / soePronAccuracy / soePronFluency / soePronCompletion`
**data-reporter.js 现有 schema**：`soeScore / soePronAccuracy / soePronFluency / soePronCompletion`

**决策**：按 ADR-006 和现有 schema，**不改字段名**。理由：
1. B020 教训：前端字段名必须 1:1 对齐飞书列名，否则 FieldNotFound
2. ADR-006 是权威设计文档，飞书表列早已按此命名
3. 任务描述里的字段名与 ADR-006 不一致，应以 ADR 为准

**⚠️ 待用户决策**：如果飞书 `app_behavior_log` 表列名现在确实是 `accuracy/fluency/completion/pronAccuracy`（非 soeXxx），需要用户在飞书后台截图确认；此时我将把 data-reporter 字段名改为与飞书完全一致。目前我按 ADR-006 执行。

## 7. 风险与待决策

### 最大风险：真机未测试
- iOS 12 Safari 的 `ScriptProcessorNode.createScriptProcessor(4096,1,1)` 在 iPhone 6 Plus 上可能偶发卡顿
- `MediaStream` 必须 HTTPS（已启用 _cert.pem）
- 第一次调用需弹权限对话框，学生可能点拒绝 → 已处理，失败返回 null
- 需要真机跑一次才能确认端到端链路

### 待决策
1. **字段名对齐**：按 ADR-006（soeXxx）还是任务描述（xxx）？我已按 ADR-006 执行，等用户确认飞书列名后如需改名是几行 sed 的事
2. **录音按钮交互**：当前是"点击开始 → 再点停止"两段式，不是按住说话。对三年级学生来说两段式更稳（iPhone 6 Plus 触控精度）
3. **学生自评是否保留**：B 录完还要点"说得好/再练练"才触发判分。SOE 分数进研究数据，自评仍决定是否触发支架。确认是否需要改为按 SOE 分数自动判分（比如 soeScore < 60 → 触发支架）
4. **录音时长上限**：当前未设上限，理论上可以录到 AudioContext 挂掉。建议加 30s 上限，但需用户确认
