## 4. 统一支架 `appendSpeakStage` 设计

### 4.1 签名

```js
appendSpeakStage(parent, refText, role, {
  hint:     '提示文字（默认"大声说出来"）',
  doneLabel:'完成按钮文字',
  onDone:   function(){ /* 录完或跳过后调用，外部在此触发 onStepBComplete */ }
});
```

### 4.2 行为

1. 在 parent 下追加一个 `.coop-speak-stage` div（黄色虚线框 + 提示 + 引用文本 + 录音按钮 + 跳过按钮）
2. 录音按钮通过 `attachSoeRecordButton` 接 P1-9 SOE 链路，分数写入 `window._pendingSoeScores[role]`
3. 跳过按钮显式写 `_pendingSoeScores[role] = null`，然后调用 `onDone()`
4. 防重复：parent 里若已有 `.coop-speak-stage` 直接返回，不重复插入
5. 自动 `scrollIntoView` 到该阶段，避免被 iPhone 6 Plus 视口截断（B010 同款修法）

### 4.3 兜底

- SOE 失败：底层 `attachSoeRecordButton` 已三层兜底（getUserMedia 拒绝 / stop 失败 / 评分 10s 超时），全部返回 null 不阻塞
- 学生不会说：跳过按钮写 null 分数后照常 `onDone()`
- 引用文本无英文：listen_scenario 会先 regex 检测，无英文字母直接跳过开口阶段（防止 `aVal` 是中文时让学生念中文）

## 7. 已知约束 & 待 Merry 决策

### 7.1 「念什么」的妥协

| 题型 | 当前选择 | 备选 | 备注 |
|---|---|---|---|
| flip_match | 念全部 pair words（如 "bear, bird, horse"） | 仅念最后一对 | 选全部更接近 Output 闭环，但有 4 个词时较长。可改为 1 个。 |
| listen_scenario | 念 `aVal`（A 听到的英文动物） | 念 B 的中文判断 | aVal 是英文，B 选项是中文，让 B 念英文更符合 Output。中文场景已 regex 跳过。 |
| listen_sort | 念整段 sequence（最多 4 个词） | 念第一个 / 中文翻译 | 整段更符合 Output Hypothesis；4 词以内三年级可接受。 |

### 7.2 与 P1-9 旧实现的并存

B019 内联的 3 处（listen_relay B / listen_judge B / word_relay B）未重写为 helper。优点：稳；缺点：缺"我不会说"按钮。建议下一轮迭代统一收敛。

### 7.3 Self-efficacy 影响

加念词阶段会延长每题耗时（约 +5~10 秒），可能让 i+1 难题对低水平学生压力变大。已通过"我不会说"兜底缓解，但需要真机验证小学生使用率。

### 7.4 SOE 计费

P1-9 baseline 6 个录音点 → 本次扩展到 15 个 SOE 调用点，单次答完 4 模块 12 题约触发 12-24 次 SOE 调用。腾讯云 SOE 单价约 ¥0.0015/次，单学生单课 ¥0.04 内可控。

## 8. 待主窗口决策的最大问题

**SOE 单次评分按住式 vs 两段点击式 的体验对比：** 当前 `attachSoeRecordButton` 是"点击开始 → 再点停止"两段式。三年级学生在压力（同伴注视 + 跳过按钮诱惑）下，可能误点一次按钮就跳过。是否需要：
- 加 1.5s 防误触延迟？
- 跳过按钮初始隐藏，3 秒后才出现？
- 跳过算 null 但飞书额外标记 `soeSkipped=true` 字段？
