# ADR-005: 游戏化系统的理论根基与重设计计划

## 状态
已采纳（2026-04-11）—— 待重设计

## 背景
项目里有四个游戏化模块：成就（achievements）、宠物（pet-system）、每日任务（daily-quests）、进度地图（progress-map）。当前 UI 按钮被隐藏（`display:none`），代码仍加载但用户看不到。

清理审计时一度被判定为"看起来废弃可移到 _legacy/"，但研究者纠正：这些系统**和六理论中的两个有强关联**，不能当作可有可无的装饰删除。

## 决策
**保留所有四个游戏化文件，不移动、不删除。** 同时列入下一阶段任务：基于理论框架重新设计这些系统，让它们的设计意图能被清晰映射到对应理论。

## 理论依据

| 系统 | 对应理论 | 学者依据 | 设计意图 |
|------|---------|---------|---------|
| achievements | Self-efficacy | Bandura (1977) | Mastery experiences——通过成就解锁累积"我能做到"的成功经验 |
| pet-system | 3P (Play-based Learning) | Hirsh-Pasek et al. (2009) | Joyful learning——情感投入、内在动机 |
| daily-quests | Self-efficacy + 3P | Bandura + Hirsh-Pasek | 小目标积累 + 游戏化结构 |
| progress-map | Self-efficacy | Bandura (1977) | 可视化进步路径，强化"我在变好"的效能感知 |

## 否定的方案
- **删除游戏化系统**：会切断 Self-efficacy 和 3P 两个理论在 App 上的具体实现，答辩时这两个理论变成"只有口号没有实例"。
- **移到 _legacy/ 暂存**：等于事实性废弃，未来重新激活时还要回头找代码，不如留在原位等重设计。

## 当前问题（待重设计原因）
研究者本人指出："**现在的设计还不够把理论体现出来**"。具体表现：
- 成就触发条件随意，没有针对"掌握经验"的精心设计
- 宠物系统和学习内容割裂，是单纯娱乐而非"play-based learning"
- 每日任务没有递进难度，无法构建效能感
- 进度地图只是显示位置，没有强化"我在进步"的感知

## 重设计原则（下一阶段执行）
1. **每个游戏化元素必须能回答一个问题**：它在帮学生体验哪个理论的哪个机制？
2. **不能干扰实验数据**：奖励机制不能诱导学生"为奖励而答题"，影响 construct validity
3. **和合作冒险主线整合**：游戏化不是另起炉灶，而是嵌入现有题型流程
4. **可量化效果**：重设计后要能在数据里看到 self-efficacy 量表分数的变化

## 论文怎么写
The app integrates four gamification modules grounded in Self-efficacy theory (Bandura, 1977) and Play-based Learning (Hirsh-Pasek et al., 2009). Achievements provide structured mastery experiences, the pet system fosters joyful engagement, daily quests build incremental success, and the progress map visualizes learner growth. These elements are designed to operationalize theoretical constructs rather than serve as superficial rewards.

## 后果
- 短期：保留代码占用约 48KB，不影响性能
- 中期：进入下一阶段任务清单，需要重设计
- 长期：答辩时 Self-efficacy 和 3P 理论不再"只有理论没有实例"
