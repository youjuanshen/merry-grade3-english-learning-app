# ADR-011：站内错题重做与通关机制

## 状态
**已通过（2026-04-12）**

## 背景
学生做完一个关卡（5道题）后，可能有些题答错过（靠支架帮助才做对）。需要验证学生是否真正掌握。

## 决策

### 错题重做机制
1. 做完站内所有题后，所有曾经答错过的题进入"错题库"
2. 错题库的题重新出现，支架机制照常运作（错了仍然会触发支架提示）
3. 判定标准：该题本次作答中**一次性直接答对（没有触发错误/支架）** = 该题算掌握，从错题库移除
4. 如果重做时又错了（哪怕靠支架最后做对了），该题继续留在错题库
5. 错题库全部清空 = 通关成功
6. **通关评语和错题重做评语均采用多样化评语系统**：通关时从多条鼓励语中随机选取（mastery experience 强化），错题重做答对时给予专门的肯定评语（区别于普通答对），避免固定文案被学生忽略

### 不设保底机制
暂不设置轮次上限或时间上限。理由：当前担心的问题是学生做太快而非太慢。

### 话术
- 进入错题重做时显示："还有X道再练一次就通关啦！"
- 不说"你错了X道"

### 数据标记
- 重做的题标记 `isRetry: true`，与首次作答区分
- 用于后续分析"scaffolding fading效果"

## 理论依据

- **ZPD + Scaffolding（Vygotsky + Wood et al. 1976）**：支架的最终目标是fading——学生从"需要帮助才能做对"到"独立完成"。错题重做验证的就是这个过程。
- **Self-efficacy（Bandura）**：通关成功 = mastery experience，是自我效能感最强的来源。冒险地图可视化进度进一步强化。
- **CLT（Cognitive Load Theory, Sweller）**：固定不变的评语会被学生习惯性忽略，变成无效的 extraneous load；情境化多样评语（根据答对/答错/连续答对/错题重做答对/通关等场景给不同评语）让反馈变成有意义的 germane load，帮助学生主动加工学习体验。
- 注意：此机制**不是** i+1（Krashen）。i+1管的是输入难度选择，不是重做机制。

### 补充改进（专家团二次审查）
- **记录每题错误次数**：stationProgress 中为每道题维护 errorCount 计数器，用于后续分析 scaffolding 有效性（错误次数越少的题，重做时一次通过率是否越高）
- **错题重做时打乱顺序**：错题库的题重新入队时 shuffle，排除学生靠位置记忆答对的可能，提高研究数据可信度
- **错题阶段进度条改文案**：进入错题重做阶段后，进度显示从"第X/5题"改为"错题 X/N"，让学生清楚知道还剩几道
