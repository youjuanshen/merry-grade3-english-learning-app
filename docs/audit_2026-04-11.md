## 建议修复顺序

1. **先修 P1 #1（reading/s3 硬崩溃）** —— coop-types.js:2270 的硬编码 `chineseOrder`。这是阻断整站的 blocker，而且未来任何 warmup pairs>4 的站点都会踩到。
2. **清理 listening/s1 和 reading/s1 的 cat/tiger 超纲题** —— 这是最刺眼的 construct validity 漏洞（论文里拿不出去）。要么改成 monkey/duck（本教材已学），要么直接删除。
3. **重写 reading/s4 的 3 道情境阅读题** —— stepA 文本全部超纲（颜色词+has/long/legs），本站 3 题全废。这个改动会影响理论对齐，需要和教师角色一起讨论。
4. **清理 writing/s4 / speaking/s2 / speaking/s3 的零散超纲词**（beautiful / Cool! / red）。
5. **统一 assets/images 图片分辨率**（bear/duck/monkey 从 640 下采样到 300）。
6. **把 cat.png / tiger.png 从 assets/images 移除**（或至少从代码引用里剔除干净后再考虑）。
7. **把听写排序/翻牌/句子排序三类非 stepAB 题型的 schema 在 docs 里明确标注**，避免后续审计和补题时混淆。
