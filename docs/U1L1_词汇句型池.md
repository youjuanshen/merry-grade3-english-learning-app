# U1L1 可用词汇句型池（project-based 剧本设计专用）

> 本文件是 U1L1 所有合作题型（含 project-based）的词汇和句型**天花板**。任何超出本文件范围的词/句型一律视为超纲，禁止使用。
> 最后更新：2026-04-12（由 agent 自动生成）
> 来源：`docs/知识点清单_U1L1.md` + `data/u1l1_coop.js`
>
> **审计方法**：先抽取知识点清单"✅ 合法词汇/句型"白名单，再 grep 全量题库，统计每词出现次数，并交叉比对清单与题库差异。
>
> **绿灯**：清单有 + 题库用过 → 100% 可用
> **黄灯**：清单有 + 题库没用 → 可用但需谨慎（缺真实语境参照）
> **红灯**：黑名单 → 绝对禁用
> **❓**：题库用过但清单未明确列 → 待用户拍板

## 5. 已确认的 Greetings / Discourse markers

> **重要发现**：题库 **完全没有** hello / hi / ok / yes / no / thank you / goodbye / bye 等任何寒暄用语的英文出现，知识点清单也没列。

| 词 | 清单 | 题库 | 状态 |
|---|---|---|---|
| Hello / Hi | ❌ | ❌ | **禁用** |
| OK / Yes / No | ❌ | ❌ | **禁用**（剧本 A/B 回应只能用动作 + 中文提示，不能说 "Yes" / "No"） |
| Thank you | ❌ | ❌ | **禁用** |
| Goodbye / Bye | ❌ | ❌ | **禁用** |

> **剧本设计含义**：U1L1 的剧本**无法用英文做问候或回应**。建议 A/B 之间的互动只靠"指物 + This/That 句 + It's/It can 句"的指认—描述循环推进，不要写对话式寒暄。

## 📌 剧本设计硬性约束（给后续 agent 的）

1. **所有英文台词只能从第 4 节的 6 种句型里取**，每句话必须能在本文件找到对应模板
2. **A/B 之间没有英语寒暄**，互动靠"指认→描述"推进，不写 Hi/Yes/No/Thank you
3. **场景/角色只能写在中文字段**（scenario / role 的中文部分），英文台词里不提 zoo/teacher 等
4. **monkey/duck 只做次要角色**，不让学生反复造句；主力句型练 bear/horse/bird/panda/rabbit
5. **fast 只以 `can run fast` 形式出现**，不要写 `It's fast.`
6. **`They are ___.` 只用 birds 版本**，其他复数保守
7. 任何想新增的词必须先回到 `docs/知识点清单_U1L1.md` 白名单追加并经用户确认
